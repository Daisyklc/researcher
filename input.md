# 调研需求示例：实时协作白板

## 背景
我们想做一个支持多人实时协作的在线白板（whiteboard），需要调研既有的开源与闭源方案，
提炼可参考的架构设计与工程实践。

## 核心诉求
- 多人实时协作、光标同步、离线可用后再合并
- 冲突解决：优先考虑 CRDT / OT 等无冲突合并方案
- 前端画布渲染性能（大量图形对象仍流畅）
- 后端同步服务、房间/权限模型

## 技术栈倾向
- 前端：TypeScript / React / Canvas or WebGL
- 同步：WebSocket / WebRTC
- 数据结构：CRDT（如 Yjs / Automerge）

## 期望产出
- 找到若干【开源】参考项目并逐个分析其架构与可借鉴点
- 记录若干【闭源/商业】产品（名称 + 链接 + 简介）作为对标
